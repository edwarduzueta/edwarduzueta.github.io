Software engineering is often misunderstood as simply the act of writing code, especially when a course focuses on web application development. However, this class made it clear that web development is only one way to practice much broader software engineering principles. Through hands-on projects and structured workflows, I learned that software engineering is really about how software is planned, built, managed, and maintained over time, as well as the responsibility engineers have for the systems they create.
Agile Project Management and Issue-Driven Development

One of the most important ideas I learned was Agile Project Management. Agile project management is an approach to organizing work that values flexibility, continuous improvement, and responding to change instead of strictly following a fixed plan from start to finish. Rather than attempting to predict every detail at the beginning of a project, Agile encourages teams to break work into smaller pieces, deliver results incrementally, and adjust as new information becomes available.

Within Agile, we specifically used Issue Driven Project Management. This is a method where all work is represented as issues, such as bugs, features, or tasks, that are tracked in a shared system. Each issue clearly defines what needs to be done and allows progress to be visible to everyone involved. This system helped keep projects organized and made collaboration much easier.

While we applied this approach to web applications in class, I can easily see myself using it in other contexts. For example, Issue Driven Project Management could be used for coordinating a group research project, organizing tasks for a military planning exercise, or managing development for non-web software such as desktop or embedded systems. Any project that involves multiple tasks, changing requirements, and teamwork could benefit from this Agile style.
Configuration Management and Controlled Change

Another core concept I learned was Configuration Management. Configuration management refers to the process of systematically tracking, controlling, and documenting changes to software and related files. This includes managing different versions of code, recording who made changes and why, and ensuring that everyone is working with consistent and correct files.

Before this course, I underestimated how important this process was. Configuration management prevents chaos as projects grow larger and more complex. It allows teams to collaborate safely, recover from mistakes, and understand the history of a project. Beyond web development, configuration management is essential in areas like cybersecurity, military systems, and large-scale software infrastructure, where errors can have serious consequences. Learning this concept helped me understand that discipline and organization are just as critical as technical skill.
Ethics in Software Engineering

The course also emphasized Ethics in Software Engineering, which refers to the moral responsibility engineers have when creating software that affects other people. This includes protecting user data, ensuring security, avoiding misleading behavior, and considering who might be harmed or excluded by a system.

This topic stood out to me because software increasingly shapes how people live, communicate, and make decisions. Ethical concerns apply far beyond web applications, especially in fields like intelligence, defense, healthcare, and finance. This class reinforced the idea that engineers cannot ignore the consequences of their work. Writing functional code is not enough if that code causes harm, violates privacy, or is used irresponsibly. Ethics is a core part of being a professional software engineer.
Final Thoughts

Overall, this course changed how I view software engineering. While web application development was the practical focus, the deeper lessons were about structure, teamwork, accountability, and responsibility. Agile Project Management showed me how to handle complex work in flexible and realistic ways. Configuration Management taught me the importance of controlled change and discipline. Ethics reminded me that software engineers have real-world responsibilities that go far beyond the screen.

These concepts are not limited to web development. They apply to almost any technical or collaborative project I may encounter in the future. By the end of this class, I no longer see software engineering as just coding, but as a thoughtful, disciplined process that balances technical execution with human impact.
